# AlgoAudit

AlgoAudit is a React + TypeScript web app for benchmarking JavaScript DSA algorithms on generated array test cases, then visualizing and exporting results.

## Features

- Built-in algorithms: Bubble Sort, Quick Sort, Merge Sort, Insertion Sort, Selection Sort.
- External algorithm loading: paste JavaScript that returns multiple algorithm definitions.
- Single custom benchmark mode: run one custom code snippet quickly.
- Auto-estimated time complexity from measured runtime growth (not hardcoded).
- Operations tracking (when your code returns `operations`).
- Charts: bar, line, and radar views.
- Export benchmark report as `.txt`.

## How To Use

### 1) Built-in benchmark

1. Select algorithms in the left panel.
2. Select test cases in the middle panel.
3. Click `RUN BENCHMARKS`.

### 2) Single custom benchmark

1. (Optional) Set algorithm name.
2. Paste code in the custom code box.
3. Click `BENCHMARK SINGLE`.

Single mode expects JavaScript with access to `arr`:

```javascript
const a = [...arr].sort((x, y) => x - y);
return a;
```

If you want operation counts and space complexity in results, return an object:

```javascript
let operations = 0;
const a = [...arr];
for (let i = 0; i < a.length; i++) {
  for (let j = 0; j < a.length - i - 1; j++) {
    operations++;
    if (a[j] > a[j + 1]) {
      [a[j], a[j + 1]] = [a[j + 1], a[j]];
      operations++;
    }
  }
}
return {
  result: a,
  operations,
  spaceComplexity: "O(1)"
};
```

### 3) External multi-algorithm benchmark

1. Paste external code that `return`s an array or object map of algorithm definitions.
2. Click `LOAD EXTERNAL`.
3. Select loaded external algorithms from the list.
4. Click `RUN BENCHMARKS`.

Definition shape:

```javascript
return [
  {
    name: "Heap Sort",
    run: (arr) => {
      // ...
      return { result: arr, operations: 0 };
    },
    spaceComplexity: "O(1)"
  }
];
```

Also available:

- `LOAD DSA TEMPLATE`: loads a starter set of important DSA algorithms.
- `CLEAR EXTERNAL`: clears loaded external algorithms.

## Input/Execution Model

- Current datasets are numeric arrays generated by selected test cases.
- Benchmarks run in-browser with `performance.now()`.
- External and single custom code run synchronously.
- Async algorithms (`Promise`) are currently not supported.

## Metrics

- **Time (ms)**: measured wall-clock runtime.
- **Operations**: from algorithm result when provided; otherwise `0`/`N/A`.
- **Estimated Time Complexity**: inferred from measured scaling across input sizes.
- **Space Complexity**: displayed from algorithm metadata or custom return data.
- **Performance Score**: normalized relative score across current result set.

## Notes and Limitations

- Complexity estimation needs multiple input sizes; otherwise it shows `Insufficient data`.
- Results for `BENCHMARK SINGLE` replace older rows with the same custom algorithm name.
- External algorithm keys are auto-generated if omitted.
